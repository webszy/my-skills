# Risk Checklist

Use this checklist before concluding a metric changed for product reasons.

| Risk | Check |
|---|---|
| Release impact | Did a new app version ship during the period? |
| Metadata change | Did title, subtitle, screenshots, keywords, or store listing change? |
| Price change | Did subscription, IAP, trial, discount, or regional pricing change? |
| Remote Config change | Did ad, paywall, onboarding, country, version, or review config change? |
| UA budget change | Did spend, campaign mix, or bid strategy change? |
| Channel change | Did media source, campaign, adset, creative, or source type mix change? |
| Country change | Did geo mix change materially? |
| Ad strategy change | Did placement, format, frequency, mediation, or cooldown change? |
| SDK version change | Did analytics, ad, attribution, purchase, or crash SDK change? |
| Missing events | Are core success, permission, ad, paywall, purchase, or result events missing? |
| Metric conflict | Do platforms define installs/downloads/revenue differently? |
| Missing dates | Are there missing daily rows or partial export windows? |
| Attribution postback gap | Are ad revenue, IAP, or key events missing from MMP? |
| Review risk | Could monetization or permission changes affect app review? |
| Misclick risk | Is CTR unusually high or is ad placement near primary actions? |
